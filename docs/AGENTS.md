# AGENTS.md — エージェント運用ルール

仕様の正本は `docs/SPEC.md`。全エージェントはこれを基準に作業する。

## 役割分担

| エージェント | 責任範囲 |
|------------|---------|
| **Claude** | 仕様責任者・設計・レビュー |
| **Codex** | コード実装・テスト・リファクタリング |
| **Gemini** | 発想拡張・セカンドオピニオン・テスト入力生成 |

## 優先順位（競合時）
1. `SPEC.md`
2. `AGENTS.md`
3. 各エージェント専用指示ファイル

## 依頼の粒度
1回の依頼は1職種に限定する。

**良い例**
- Claude: 「MVPの画面仕様を整理して」
- Codex: 「発明生成APIを実装して」
- Gemini: 「fusion patternの代替案を20個出して」

**悪い例**
- 「設計して実装してレビューしてUIも出して」

## 実装ルール (Codex向け)
- 仕様書の変更を実装と同時に行わない
- 発想ロジックを独断で改変しない
- テストコードは `tests/` に置く

## レビュー責任
- 仕様整合レビュー: Claude
- 実装レビュー: Codex または Claude
- 発想レビュー: Gemini または Claude
- 最終仕様承認: Claude
- 最終実装承認: Codex
